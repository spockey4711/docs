## Route inventory

The public HTTP surface currently includes:

- `POST /calculate`
- `GET /calculations/recent`
- `POST /auth/register`
- `POST /auth/login`
- `POST /auth/logout`
- `POST /plans`
- `GET /plans`
- `GET /billing/status`
- `GET /billing/invoices`

## Authentication

Most product auth is session-based rather than bearer-token-based.

That means:

- login creates an `HttpOnly` session cookie
- logout clears the session
- form-based state-changing actions use CSRF protection

## Core flows

- user registration
- user login
- single-session fueling calculation
- saved plans
- billing self-service metadata and invoice history

## Notes

The app mixes JSON APIs and HTML or SPA shell routes. This reference focuses on the API-shaped routes, not the marketing or shell pages.
