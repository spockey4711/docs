Fueling Optimizer helps endurance athletes turn validated session inputs into concrete fueling targets in `g`, `ml`, and `mg`.

It is intentionally narrow. The product covers one session at a time and focuses on practical, explainable recommendations before, during, and after training or race-specific efforts.

## Product posture

Fueling Optimizer is positioned as sports science software, not an AI coach.

<Note>
  Recommendations are deterministic, transparent, and intentionally free of black-box adaptation.
</Note>

## Current scope

The active product currently supports:

- single-session fueling recommendations
- pre, during, and post-session outputs
- practical food and drink mapping
- user accounts and saved preferences
- Pro billing and entitlements
- PDF-export-oriented presentation flows

The active sports in the calculation API are `bike` and `running`.

## Instrumentation

PostHog can be enabled in both the shared SPA shell and the FastAPI backend using the same project key. The frontend uses the HTML snippet integration with a proxy `api_host` plus the PostHog EU `ui_host`. The backend uses the Python `posthog` package for successful registration, login, and calculation events.
