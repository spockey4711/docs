<Info>
  **Prerequisites**:
  - Node.js version 19 or higher
  - A docs repository with a `docs.json` file
</Info>

Follow these steps to run the Fueling Optimizer docs locally.

<Steps>
<Step title="Install the Mintlify CLI">

```bash
npm i -g mint
```
</Step>

<Step title="Preview locally">

Navigate to your docs directory where your `docs.json` file is located, and run the following command:

```bash
mint dev
```

A local preview of your documentation will be available at `http://localhost:3000`.

</Step>
</Steps>

## Repository note

This documentation lives in its own Git repository. The docs are related to Fueling Optimizer, but they are versioned separately from the application repository.

## Custom ports

By default, Mintlify uses port 3000. You can customize the port Mintlify runs on by using the `--port` flag. For example, to run Mintlify on port 3333, use this command:

```bash
mint dev --port 3333
```

If you attempt to run Mintlify on a port that's already in use, it will use the next available port:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## CLI compatibility

The local validator can be stricter than the hosted documentation examples. If a field documented on the Mintlify site fails locally in `mint dev`, prefer the locally working configuration and update the CLI before reintroducing the field.

To update the CLI:

```bash
mint update
```

## Validating links

The CLI can assist with validating links in your documentation. To identify any broken links, use the following command:

```bash
mint broken-links
```

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: docs.json contains unsupported fields'>

    If `mint dev` rejects a field that appears in the Mintlify docs, remove the field temporarily and verify your local CLI version with `mint --version`.
  </Accordion>

  <Accordion title="Issue: Port 3000 is already in use">
  
    Start the preview on a different port with `mint dev --port 3333`.
  </Accordion>
</AccordionGroup>
